# Kernel One — Constitutional Execution Kernel

**Epistemic tier:** T2 (engineering hypothesis — a working local governance kernel; not yet a proven component of the T0 core).

Kernel One is a self-contained, fail-closed execution kernel that admits or rejects
model proposals against a constitutional manifest. It is the local, single-file
embodiment of the AEGIS RALPH loop: every accepted transaction produces an
HMAC-signed, hash-chained **witness** row in a SQLite ledger; anything that fails
validation returns `RECONCILED_FALLBACK` and writes nothing.

```
READ → ASSESS → T0_CHECK → LOCK → PROPAGATE → HARMONIZE
```

## Admission gate

`KernelOneEngine.process_transaction(request, raw_model_response)`:

1. **validate_request_structure** — request must carry `request_id`, `task`, and a
   `metrics` dict with four numeric observables (`uncertainty`, `tool_failures`,
   `novelty`, `reviewer_disagreement`). Missing `request_id` is rejected — it is the
   audit anchor.
2. **calculate_residual_delta** — weighted entropy of the metrics. `delta ≥ 0.70`
   (`safety.delta_critical` in `INDEX.yaml`) → rejected.
3. **validate_proposal_schema** — the model output must be exactly
   `{plan, output, tool_calls}`; any `target_file` or tool-call touching a file that
   is unregistered or `frozen: true` in the manifest is rejected.
4. On success: compute `task/plan/output/index` hashes, derive a `witness_id`,
   HMAC-sign it, and record `witnesses` + `artifacts` rows.

The manifest (`INDEX.yaml`) is the admissibility boundary: any mutation outside it
is invalid.

## Run it

```bash
python init_db.py            # create memory_store.sqlite (schema only; gitignored)
python test_kernel_one.py    # 9/9 adversarial assertions must pass (exit 0)
```

`memory_store.sqlite` is a runtime witness ledger, **not** source — it is
regenerated by `init_db.py` and excluded via `.gitignore`.

## Signing key — ROTATE BEFORE PRODUCTION USE

The witness signing key is read from the environment:

```python
KERNEL_SECRET_SIGNING_KEY = os.environ.get(
    "AEGIS_KERNEL_SIGNING_KEY", "aegis_sovereign_secure_key_vector_2026"
).encode("utf-8")
```

The default is the historical value from the recovered source and is **public in git
history** — it authenticates nothing on its own. Set `AEGIS_KERNEL_SIGNING_KEY` to a
secret value in any real deployment; witnesses signed under the default must be
treated as unauthenticated.
